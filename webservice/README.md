# URL Shortener Web Service

A simple yet complete URL shortener service built with Rust, featuring JWT authentication, in-memory storage, OpenAPI documentation, and a clean REST API.

## Features

- 🔐 **JWT Authentication** - Secure user registration and login
- 🔗 **URL Shortening** - Generate 7-character alphanumeric short codes
- 🗄️ **In-Memory Storage** - Fast, async-safe storage using `tokio::sync::RwLock`
- 🔒 **Protected Endpoints** - Create and manage URLs with bearer token authentication
- 🚀 **Public Redirects** - Fast URL redirects without authentication
- 🔑 **Argon2 Hashing** - Secure password storage
- 📚 **OpenAPI/Swagger UI** - Interactive API documentation

## Tech Stack

- **axum** 0.7 - Web framework
- **tokio** - Async runtime
- **jsonwebtoken** - JWT encoding/decoding
- **argon2** - Password hashing
- **tower-http** - HTTP middleware (tracing, CORS)
- **utoipa** - OpenAPI spec generation
- **utoipa-swagger-ui** - Swagger UI integration

## API Endpoints

### Authentication

#### Register a New User

```bash
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "alice",
    "password": "secret123"
  }'
```

### URL Management (Protected)

**Note:** All URL management endpoints require a valid JWT token in the `Authorization` header.

#### Create a Short URL

```bash
curl -X POST http://localhost:3000/api/urls \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "long_url": "https://www.rust-lang.org"
  }'
```

**Response:**
```json
{
  "short_code": "aBc123X",
  "short_url": "http://localhost:3000/aBc123X",
  "long_url": "https://www.rust-lang.org"
}
```

#### List Your URLs

```bash
curl http://localhost:3000/api/urls \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

**Response:**
```json
[
  {
    "short_code": "aBc123X",
    "long_url": "https://www.rust-lang.org",
    "created_at": 1718640000
  },
  {
    "short_code": "xYz456P",
    "long_url": "https://github.com",
    "created_at": 1718640100
  }

```

### URL Redirect (Public)

## Complete Workflow Example

```bash
# 1. Register a user
REGISTER_RESPONSE=$(curl -s -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"bob","password":"password123"}')

echo "Registration: $REGISTER_RESPONSE"

# 2. Extract the token (requires jq)
TOKEN=$(echo $REGISTER_RESPONSE | jq -r '.token')
echo "Token: $TOKEN"

# 3. Create a short URL
CREATE_RESPONSE=$(curl -s -X POST http://localhost:3000/api/urls \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"long_url":"https://www.rust-lang.org"}')

echo "Created: $CREATE_RESPONSE"

# 4. Extract the short code
SHORT_CODE=$(echo $CREATE_RESPONSE | jq -r '.short_code')
echo "Short code: $SHORT_CODE"

# 5. Test the redirect
curl -L http://localhost:3000/$SHORT_CODE

# 6. List all your URLs
curl -s http://localhost:3000/api/urls \
  -H "Authorization: Bearer $TOKEN" | jq
```

## Configuration

- **Server Address:** `0.0.0.0:3000`
- **JWT Secret:** `super-secret-jwt-key-change-in-production` ⚠️
- **Token Expiry:** 86400 seconds (24 hours)

⚠️ **Security Warning:** Change the JWT secret before using in production!

## Error Handling

The API returns appropriate HTTP status codes:

- `200 OK` - Successful request
- `303 See Other` - URL redirect
- `400 Bad Request` - Invalid URL format
- `401 Unauthorized` - Missing or invalid JWT token
- `404 Not Found` - Short URL not found
- `409 Conflict` - Username already exists
- `500 Internal Server Error` - Server-side error

Error responses include a JSON body:
```json
{
  "error": "Error message here"
}
```
